# ghc


Most successful installations use older versions of GHC. If you have a newer GHC version, you can try using the “–lib” flag:

```bash
cabal install --lib --allow-newer
```

Based on [troubleshooting](https://www.euterpea.com/troubleshooting) page, I tried higher Versions:
- [ ] [9.6.6](9.6.6) just to a higher version than current Euterpea ghc version of `9.4.6`
- [ ] [9.12.4](9.12.4) to put the version as close as `clash-lang` compiler version
